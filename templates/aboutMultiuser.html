Multi-User System

👨‍🎓 Student Panel

Students can register and log in to mark attendance, view attendance history, and receive OTPs or location-based prompts depending on their class setup. Each student is linked to a specific subject and institute.

🏫 Institute Manager Panel

Managers can create subjects, register students, manage attendance sessions, and control attendance methods (Face, OTP, Location). Each manager operates their own independent institute dashboard.

🔐 Admin (Superuser)

The admin can monitor multiple institutes, verify manager accounts, analyze system usage, and resolve disputes. This role has full access across all institute-level activities for moderation and auditing.

🔄 Scalable User Roles

The system is built to scale — allowing more roles such as teaching assistants, staff, or department heads to be integrated in future releases. Each user’s access is isolated based on their role and institute.

🛡️ Role-Based Security

All users access features based on their permissions. For example, students can’t view other students’ data, and managers can't access institutes they don't manage. This maintains clean, secure boundaries.
